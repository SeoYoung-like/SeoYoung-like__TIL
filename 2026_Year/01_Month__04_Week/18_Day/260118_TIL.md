## 오늘

* 일정 : JavaScript 고수 연합 - 스터디

* 목표 : 코어 자바스크립트 - 1단원 진행

* 내용:

  * Javascript - Private 변수랑 referenct 변수 차이점 이해 (완)
  * 변수의 메모리 사용법 이해

  ---

  * 원시 타입 : 불변성, 식별자 메모리 - 데이터 메모리 2가지 사용
  * 참조 타입 : 가변성, 식별자 메모리 - 객체 메모리 - 식별자 메모리 - 데이터 메모리 4가지 사용
  * 얕은 복사, 깊은 복사 - 사용법 익히기
  * undefined, null, empty 차이점 이해
    * undefined - 선언시 자동 할당 ( 할당 안하면 자동 생성 )
    * null - 선언시 의도적 할당 ( 할당 하지 않았다는 것을 의미 )
      * [주의점] object 타입 - javascript의 고질적인 오래된 버그
    * empty - array에서 변수 할당없이 길이를 늘릴 경우 empty 자동할당
      * [주의점] undefined와 null 과 다르게 사용시 아예 접근 불가(누락 시킴)
        즉, reduce, foreach 문이든 접근이 아예 안됨

  ---

  * EMACScript - 표준화 자바스크립트
    * ES6 : 본격적으로 프로그래밍 언어의 기틀을 다짐
      * 화살표 함수, let const 등
    * ES7 :  async await 등장

  ---

  * node.js : 브라우저 없이도 자바스크립트 사용가능
    * [주의점!] node.js로 사용시 브라우저 자체에 있는 WebAPI 사용 불가
    * 대신 filesystem, http/https api등 다른 서버나 로컬 컴퓨터에 필요한 api를 사용할 수 있음

  ---

  * 웹 페이지가 브라우저에서 작동되는 과정
    * html 로드 => html 파싱 => DOM 트리 생성중
    * css 로드 => css 파싱 => cssom (css object model) 생성중
      * 렌더링 엔진(Webkit) 사용
    * JS 로드(html, css 렌더링 중단) => JS 파싱 => JS 실행 => Syntax tree 생성
      * 자바스크립트 엔진(Webkit) 사용
    * DOM 트리 재개생성
    * cssom (css object model) 재개생성
    * rentertree 생성 = DOM 트리  + CSSOM
    * 디스플레이
  * JavaScrip를 Body 태그 닫히기 전에 두는 이유 
    * 위에 생성 방식에 서알 수 있 듯 'DOM 생성시 에러 발생을 방지 + 빠른 생성을 통한 사용자 친화'를 위함이다.
